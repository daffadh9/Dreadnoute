
# DreadNoute MVP Development Plan
Author: Daffa Dhiyaulhaq Khadafi

## Goal
Build the first functional version of DreadNoute that proves:
- users visit
- users interact
- users return

MVP focuses on four core systems:
1. Ghost Archive
2. Real or Fake
3. Ghost Cards (G‑Collector)
4. DreadCoin

---

## Phase 1 — Platform Foundation

Build:

- authentication
- user profiles
- database schema
- design system
- basic navigation layout

Database tables:

profiles  
ghost_archive  
investigations  
ghost_cards  
wallets  

---

## Phase 2 — Ghost Archive

Purpose:
Acquire users via searchable horror knowledge.

Features:
- ghost entity page
- search
- tags
- categories
- related entries

Outcome:
Users discover DreadNoute via search and exploration.

---

## Phase 3 — Real or Fake

Purpose:
Create user participation.

Features:
- upload media
- voting system
- comment discussion
- investigation result page

Outcome:
User‑generated content begins.

---

## Phase 4 — Ghost Cards (G‑Collector)

Purpose:
Retention system.

Features:
- card rarity
- card unlock
- card collection page
- profile showcase

Outcome:
Users return to collect cards.

---

## Phase 5 — DreadCoin Economy

Purpose:
Platform economy.

Features:
- wallet
- reward system
- card packs
- collectible unlock

Outcome:
Foundation for monetization.

---

## MVP Success Indicators

Users: 1k–10k  
Returning users  
First internal economy activity
